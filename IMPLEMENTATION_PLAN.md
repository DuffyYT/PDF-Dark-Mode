# PDF Dark Mode Extension — Implementation Plan

## Goal
Turn current distribution (~30k WAU) into a durable business by adding owned channels, measurement, and premium value while preserving user trust.

## Strategy
1. Build owned assets first (website + email list).
2. Add privacy-friendly product analytics for decision-making.
3. Introduce a Pro tier without degrading the free experience.
4. Expand into student/productivity workflows after validating demand.

## Phase 1 — Foundation (Owned Distribution + Data)

### 1. Branding and Positioning
- Define one-line value proposition for homepage/store listing.
- Refresh screenshots, extension description, and trust messaging.
- Align naming across extension, repo, and website.

### 2. Landing Page + SEO Funnel
- Launch extension landing page (main domain or dedicated domain).
- Add pages/posts targeting intent keywords:
  - "pdf dark mode edge"
  - "dark mode for pdf browser"
  - "read pdf at night"
  - "eye strain pdf"
  - "pdf reader dark theme"
- Add install CTA, feature explainer, FAQ, changelog, and privacy section.

### 3. Email Capture
- Add "Get updates" flow on website and optional in-extension entry point.
- Offer: changelog alerts, beta access, feature voting.
- Set up basic email sequence:
  - Welcome
  - Feature tips
  - New release updates

### 4. Privacy-Friendly Analytics
- Track:
  - DAU/WAU
  - 7-day and 30-day retention
  - Feature/settings usage
  - PDF source context (high-level, non-sensitive)
  - Session length buckets
- Publish clear data collection disclosure and opt-out path.

## Phase 2 — Monetization (Freemium + Pro)

### 1. Pro Packaging
- Keep core dark mode free forever.
- Pro feature candidates:
  - Custom themes
  - Scheduled dark/light switching
  - Sepia/eye-care modes
  - Brightness/contrast tuning
  - Per-site preferences
  - Shortcut customization
  - Settings sync
  - AMOLED mode

### 2. Pricing
- Test simple plans:
  - Monthly: $2–5
  - Lifetime: $15–30
- Start with lifetime + one recurring plan (avoid pricing complexity).

### 3. Conversion Flow
- In-product upgrade prompts only at relevant feature touchpoints.
- No intrusive popups; no forced signups for free usage.
- Add clear "what is free vs Pro" comparison section.

## Phase 3 — Student/Productivity Expansion

### 1. Student-Oriented Features
- Prioritize by demand and implementation cost:
  - Study mode / focus mode
  - Reading progress
  - Keyboard navigation
  - Highlight export
  - Side notes
  - AI summaries
  - Citation extraction

### 2. Product Position Shift
- Evolve messaging from "dark mode extension" to "PDF reading productivity tool".
- Validate each feature with usage + retention impact before broad rollout.

## Phase 4 — Scale and Optional Exit Paths

### 1. Cross-Promotion Engine
- Use extension + email list to launch adjacent products:
  - AI PDF assistant
  - Note-taking/study tools
  - Citation/reading workflow tools

### 2. Partnerships / B2B
- Explore integrations with e-learning, accessibility, and PDF SaaS tools.
- Pilot lightweight enterprise controls if requested by teams.

### 3. Sellability Readiness (Optional)
- Maintain metrics dashboard covering growth, retention, conversion, and reviews.
- Keep documentation clean (roadmap, changelog, privacy, support SLAs).

## Execution Principles
- Protect trust: no ads, no aggressive prompts, no friction on core utility.
- Ship in small increments with measurable outcomes.
- Prioritize features that improve comfort + productivity for frequent PDF users.
